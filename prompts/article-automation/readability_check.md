# Readability Check Sub-Agent Prompt

## Goal

作成済みの記事、画像、サイト表示を読者目線で確認し、公開前に直すべき問題を見つける。

大幅な書き直しより、品質リスクを減らすことを優先する。軽微な問題はその場で直す。

## Must Check

- 高校生でも読めるか
- 1文が長すぎないか
- 難しい言葉に説明があるか
- 見出しの順番が自然か
- テーマに関係ない固定見出しが入っていないか
- タイトルと本文が一致しているか
- 参考リンクが個別URLになっているか
- Polymarket/Metaculusが主役になっていないか
- 投資助言に見える表現がないか
- 予測市場や株式市場に関係ない記事へ、市場セクションを無理に入れていないか
- 直近10記事とテーマが重なりすぎていないか
- 直近7記事で同カテゴリが3本以上続いていないか

## Must Check Display

- 個別記事ページでタイトルが重複していないか
- スマホで横スクロールが出ないか
- `<picture>` でPC用/スマホ用画像が切り替わるか
- 最新記事にアイキャッチが出るか
- `og:image` が設定されるか
- SVG内の文字が小さすぎないか
- PNGを使う場合、重すぎる画像や表示崩れになっていないか

## Must Check Files

- 記事は `articles/` にあるか
- 本文内画像とアイキャッチ画像は `web/public/images/articles/` にあるか
- `web/dist/`、`web/node_modules/`、`web/.astro/`、`web/package-lock.json` などの生成物がcommit対象に入っていないか
- 変更ファイルだけがcommit対象になっているか

## Fix Immediately

- 誤字
- 長すぎる文
- 不自然な見出し
- テーマと関係ない固定セクション
- 投資助言に見える表現
- リンク表記
- SVGの幅や文字サイズ
- PNGアイキャッチの表示設定

## Output

自由に書いてよいが、最低限これを含める。

```md
## Readability Check Result
- 文章:
- SEO:
- 重複回避:
- 投資助言リスク:
- 表示:
- 画像:
- 修正した内容:
- 残る注意点:
```
