# Daily Article — Probable Press

## ミッション

その日の `data/` から、ニュースと予測市場（Polymarket）の間の「ギャップ」を発見し、記事化する。読者は経済・国際情勢を専門にしない一般読者。**高校生でも理解できる文体**で書く（＝高校生向けではなく、誰でも読める平易さ）。

## 実行手順

1. **データ読込**：本日の日付（UTC）に対応する以下を読む
   - `data/markets-YYYY-MM-DD.json`（必須）
   - `data/news-YYYY-MM-DD.json`（必須）
   - `data/guardian-YYYY-MM-DD.json`（あれば）
   - `data/metaculus-YYYY-MM-DD.json`（あれば）
   - `data/dune-YYYY-MM-DD.json`（あれば）

2. **ストーリー候補抽出**：
   - 同一テーマで時間軸が異なる市場の確率差（例：4月 vs 5月）
   - ニュースの論調と市場価格の温度差
   - 24h出来高が大きい（>$1M）市場と関連ニュースの対応
   - スポーツ市場（`is_sports=true`）は除外

3. **執筆**：1〜2本（無理に2本にしない）。1本目は最重要ストーリー。

4. **保存**：`articles/YYYY-MM-DD-{slug}.md`
   - slug：英数字小文字とハイフンのみ、テーマを表す3〜5語
   - 例：`2026-04-28-uae-opec-oil-100.md`

## 記事フォーマット

```markdown
# {検索意図に合わせたタイトル：50〜60字以内、キーワードを前置}

_YYYY-MM-DD | カテゴリ：{エネルギー/国際/金融政策/etc} | Gap：{ギャップの一文要約}_

{リード段落：140字以内、本文を要約。meta descriptionに自動抽出される}

---

## {見出しはキーワード入り}

{本文：1段落あたり2〜3文以内。専門語は出した瞬間に定義する}

---

## 予測市場のスコアカード

| マーケット | 現在の確率 | 24時間出来高 |
|---|---:|---:|
| 市場名（日本語訳） | **XX.X%** | XXX万ドル |

---

## まとめ：{N}つのポイント

1. ...
2. ...

---

## ソース

- NHK経済: [タイトル](URL)
- Guardian: [タイトル](URL)
- Polymarket: [event名](URL)
```

## 文体ルール

- **段落は短く**：1段落2〜3文。長い段落は分割
- **専門語は即定義**：「OPEC（石油輸出国機構）」のように初出で説明
- **能動態**：「〜される」より「〜する」
- **具体的な数字**：「価格が上がった」ではなく「1バレル100ドルを超えた」
- **比喩は控えめに**：高校生でも分かる比喩は使うが、過度な擬人化は避ける
- **断定は事実のみ**：解釈・推測には「とされる」「可能性がある」を使う
- **絵文字は使わない**

## ギャップの判定

- **|gap| ≥ 10pp** を「🚩 flagged」相当とする（記事タイトル・冒頭で強調）
- **時間軸スプリット**（同一テーマで月をまたぐ確率差が25pp以上）も独立した編集ネタとして扱う
- **gap=0pp でも編集価値あり**：ニュースと市場が完全一致 = 「合意の確認」として書ける

## ソースリンク

- 必ず一次ソースのURLを記載
- NHK の URL は `https://www3.nhk.or.jp/news/` 配下を使う（記事JSONの `url` を優先）
- Polymarket は `https://polymarket.com/event/{slug}` の形式

## 禁止事項

- 投資助言にならない表現に留める（「買うべき」等は書かない）
- 予測市場の確率を「予言」として書かない（あくまで「市場が織り込んだ確率」）
- 学習データ由来の事実を補完しない。data/ にない情報は書かない
- インサイダー断定はしない（オンチェーンデータからの示唆も「観察」レベルに留める）

## 完了後

- `git add articles/` → `git commit -m "article: YYYY-MM-DD {テーマ要約}"` → `git push origin main`
- ユーザーに記事タイトルとファイルパスを報告する
